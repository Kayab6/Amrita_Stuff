## lecture 1 - C 
### Software Development Life Cycle (SDLC)

| House Construction      | Software Development       |
| ----------------------- | -------------------------- |
| Choose land             | Select the problem/project |
| Inspect the land        | Feasibility Study          |
| Decide what to build    | Requirement Analysis       |
| Draw blueprint          | Software Design            |
| Build the house         | Coding                     |
| Check for defects       | Testing                    |
| Hand over to owner      | Deployment                 |
| Repairs after moving in | Maintenance                |



## 1. Project Selection
## Feasibility Study

Question:
"Can we actually build this project?"

There are several types.

Technical Feasibility-Do we have the technology?

Economic Feasibility

Operational Feasibility-Will people actually use it?

Schedule Feasibility-Can it be finished within time

## 3. Requirement Analysis-Now understand exactly what the user wants.
-Functional Requirements-(what the software should do)

-Non-functional Requirements (how the software should behave)


## 4. Design
# High-Level Design (HLD)
Big picture.
Shows:-
Overall modules
How modules communicate
Architecture

# Low-Level Design (LLD)

Detailed design. - Includes:
-Functions
-Variables
-Algorithms
-Data structures
-Database tables (if applicable)
