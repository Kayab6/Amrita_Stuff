## PSEUDOCODE



Pseudocode is a method of writing an algorithm using simple English statements and programming constructs without adhering to the syntax of any programming language.

### BASIC SWAP PSEUDO CODE
BEGIN
    READ A, B
    TEMP ← A
    A ← B
    B ← TEMP
    PRINT A, B
END

## ALGORITM
An algorithm is a set of step-by-step instructions to solve a given problem or achieve a specific goal.


### swap algorithm 
Step 1: Start
Step 2: Read A and B
Step 3: SUM ← A + B
Step 4: Display SUM
Step 5: Stop


| Algorithm                                  | Pseudocode                                     |
| ------------------------------------------ | ---------------------------------------------- |
| Written as numbered or plain English steps | Written in a programming-like format           |
| Less structured                            | More structured                                |
| Describes the solution                     | Describes the solution in a form close to code |


Feasible Solution: A solution that satisfies all the constraints of a problem.
Infeasible Solution: A solution that violates one or more constraints.
Optimal Solution: The best feasible solution according to the objective function (such as minimum cost or maximum profit).


| **Static Memory**                                                                       | **Dynamic Memory**                                                  |
| --------------------------------------------------------------------------------------- | ------------------------------------------------------------------- |
| Allocated before program execution or at compile time (conceptually, fixed in advance). | Allocated during program execution (runtime).                       |
| Size is fixed.                                                                          | Size can grow or shrink.                                            |
| Faster access and allocation.                                                           | Slightly slower due to runtime allocation.                          |
| Cannot be resized.                                                                      | Can be resized by allocating or freeing memory.                     |
| Example: Arrays with a fixed size (`int a[10]` in C).                                   | Example: Linked lists, dynamically allocated arrays, trees, graphs. |

| C                                  | Java                                                |
| ---------------------------------- | --------------------------------------------------- |
| `malloc()` and `free()`            | `new` (no `free()`)                                 |
| Programmer manually frees memory   | Garbage Collector automatically frees unused memory |
| Explicit dynamic memory management | Automatic memory management                         |
